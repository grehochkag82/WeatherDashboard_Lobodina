# Weather Dashboard

## Краткое описание

Weather Dashboard — Android-приложение для отображения погодных данных из трёх источников (температура, влажность, скорость ветра). Приложение демонстрирует использование корутин Kotlin для асинхронных операций, параллельной загрузки данных и обработки ошибок без блокировки UI-потока.

## Функциональность приложения

- Отображение температуры воздуха (15-35°C)
-  Отображение влажности (40-80%)
- Отображение скорости ветра (0-20 м/с)
-  Вычисление индекса погоды на основе полученных данных
- Параллельная загрузка данных из трёх источников (~2 секунды вместо 4.5)
-  Индикатор прогресса загрузки
-  Кнопка ручного обновления данных
-  Автоматическое обновление каждые 10 секунд
-  Имитация ошибки сети с корректной обработкой
-  Отображение ошибок в UI без краша приложения

## Технологии и библиотеки

- **Kotlin** — основной язык программирования
- **Jetpack Compose** — декларативный UI
- **Coroutines** — асинхронное программирование
- **StateFlow** — управление состоянием
- **ViewModel** — архитектурный компонент
- **Material 3** — дизайн-система

**Ответ:** `launch` не возвращает результат, `async` возвращает результат через `await()`.

| | launch | async |
|---|--------|-------|
| Возвращает | Job | Deferred<T> |
| Результат | Нет | Да (через .await()) |

- **launch** — когда не нужен результат (сохранение в БД, отправка логов)
- **async** — когда нужен результат (загрузка данных, вычисления)

```kotlin
Пример launch
viewModelScope.launch { saveToDatabase() }

Пример async
val result = viewModelScope.async { fetchData() }.await()

Параллельная загрузка с async

viewModelScope.launch {
    val temp = async { fetchTemperature() }
    val hum = async { fetchHumidity() }
    val wind = async { fetchWindSpeed() }
    
    val temperature = temp.await()
    val humidity = hum.await()
    val windSpeed = wind.await()
}
```

### D) Что произойдёт, если не обработать исключение в корутине?

Приложение крашнется

| Вопрос | Ответ |
|--------|-------|
| Что будет без обработки? | Приложение падает |
| Как обрабатывать ошибки? | try-catch внутри launch или async |
| Зачем нужен try-catch? | Чтобы показать ошибку в UI и избежать краша |

### E) Как работает автоматическая отмена корутин?

viewModelScope автоматически отменяет корутины при уничтожении ViewModel

| Вопрос | Ответ |
|--------|-------|
| Что такое viewModelScope? | Область видимости, привязанная к жизненному циклу ViewModel |
| Когда отменяются корутины? | При закрытии экрана, вызове onCleared(), ошибке в родительской корутине |
| Нужно ли отменять вручную? | Нет, viewModelScope делает это автоматически |
